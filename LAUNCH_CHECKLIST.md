## Messaging Reminders

### Always Lead With
1. **The problem:** "Most teams hand-test 5-10 prompts"
2. **The scale:** "1440+ API calls" or "100-1000+ combinations"
3. **The proof:** Real cost ($0.07, $0.083) and quality (6.61/10, 9.16/10)
4. **The outcome:** "Deployment-ready winning prompt"

### Always Include
- [ ] GitHub link
- [ ] Real experiment results
- [ ] Cost data
- [ ] Model comparison
- [ ] Open source mention

### Tone
- **Honest:** "This is real data from real experiments"
- **Humble:** "Open source, built by a developer for developers"
- **Practical:** Focus on outcomes, not hype
- **Community-oriented:** "We'd love feedback"

## Contingency Plans

### If Hacker News Submission Gets No Traction
- [ ] Wait 2 hours, check comments
- [ ] If <20 comments by hour 4, resubmit next day with slightly different title
- [ ] Double down on Twitter/Reddit (they will definitely work)

### If Reddit Posts Get Low Engagement
- [ ] More detailed replies to comments (depth > upvotes)
- [ ] Ask for specific feedback ("What would make this even more useful?")
- [ ] Cross-link to the other Reddit post
- [ ] Point to real experiments (tangible proof)

### If Users Report Bugs
- [ ] Fix immediately if possible
- [ ] Public communication ("Working on fix, will deploy today")
- [ ] Don't disappear — quick response builds trust

### If Someone Criticizes the Methodology
- [ ] Thank them, engage honestly
- [ ] Explain design choices (why 4 eval methods, etc.)
- [ ] Ask what they'd suggest instead
- [ ] Be open to feedback (show you're not defensive)
